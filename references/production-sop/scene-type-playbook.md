# Scene Type Playbook — 场景类型策略剧本

这份文档定义了**不同戏剧功能的场景各自的设计策略**。它不是质量标准（那个是 master-checklist），而是**针对场景类型的战术手册**。

每一段 Seedance 生成之前，先判断这场戏的戏剧功能是什么，然后加载对应的策略。

---

## 场景类型分类与识别

### 分类图谱

```
用户输入 / 剧本片段
        │
        ▼
    ┌───────────────────────────────┐
    │ 1. 识别本段落的戏剧功能          │
    │ 2. 加载对应策略剧本              │
    │ 3. 按策略设计故事板 + Seedance   │
    └───────────────────────────────┘
```

### 触发词与场景类型映射

| 场景类型 | 触发信号（脚本中出现的特征） |
|---|---|
| **DIALOGUE 文戏** | 两人或以上对话、单人独白、情感对峙、秘密揭露、电话对话。关键词：说、问、答、告诉、沉默、注视 |
| **ACTION 武戏** | 身体冲突、追逐、打斗、枪战、坠落、奔跑、躲避。关键词：打、追、跑、跳、摔、撞、射击、躲 |
| **SUSPENSE 悬疑** | 慢速探索、发现异常、恐怖揭示、被追踪、隐藏、等待。关键词：看、听、走、停、发现、门开了、后面 |
| **TRANSITION 过渡** | 场景转换、时间流逝、空间移动、情绪桥接。关键词：之后、同时、另一边、路上、回到 |
| **PRODUCT 产品** | 产品展示、功能演示、品牌露出、消费场景。关键词：产品、包装、使用、品牌 |
| **ATMOSPHERE 氛围** | 纯视觉诗、环境肖像、情绪景观、B-roll 段落。关键词：城市、雨、窗、光、空镜 |

### 识别流程

1. 从用户输入/剧本中提取当前段落的内容
2. 扫描动作词、对话标志、环境描述、道具出现
3. 与触发信号表匹配，确定主类型
4. 如果一段内混合多种功能（如"追跑+最后对峙"），按主导功能分类，次要功能在策略中作为变体处理

### HYBRID 混合段落规则

如果一段同时包含两种需要不同节奏的戏剧功能，不要用一个类型吞掉另一个类型。

Examples:

- 追逐 + 台词揭露：拆成 ACTION 段和 DIALOGUE/SUSPENSE 段。
- 爆炸 + 悼念：拆成 ACTION 冲击段和 ATMOSPHERE/DRAMA 余韵段。
- 产品展示 + 情绪表演：先 PRODUCT 感官段，再 DIALOGUE/ATMOSPHERE 反应段。

Priority:

- `realism-clean-frame-rhythm-gate.md` and `prompt-self-review-checklist.md` override scene-type playbook defaults.
- Short dialogue cross-cutting of 5-8 shots only applies to explicit rapid argument, comedy timing, or actionized verbal exchange.
- It does not apply to fear, silence, grief, interrogation, moral pressure, military command hesitation, or any beat that needs breath.

## DIALOGUE 文戏策略

文戏的核心是**人与人之间发生了什么**。不是台词——是台词之间的东西。

### 故事板设计

**俯视动线图重点:**
- 人物起始位置和距离（两人距离 1.5m = 亲密/对峙，3m = 社交距离，5m+ = 疏离/权力差）
- 在这场对话中人物是否移动？如果移动——谁靠近谁？（靠近 = 进攻或亲密，远离 = 防御或失望）
- 180 度轴线标注（两人之间的连线，摄影机不越线——除非权力发生了转移）
- 如果有第三者——第三者的位置是威胁还是旁观？

**机位设置:**
- 主镜头（master）: 广角，展示两人的空间关系和距离
- 过肩（OTS）: A 的肩看 B，B 的肩看 A。过肩不只是看对方——通过肩的占比表达权力。A 的肩占画面 30% = A 有权力。A 的肩只占 10% = A 在弱势。
- 单人近景（single CU）: 一个人的脸。什么时候切到单人？当她说了谎。当她被击中。当她在决定。
- 反应镜头优先于说话镜头。说话的人说了什么不重要——听的人怎么反应才重要。

**镜头数策略:**
- 对白节奏决定镜头数，不设固定公式
- 长对白: 3-4 个机位，缓慢切换。让对话有时间落地。切换在"一句话的重量沉下去之后"。
- 短交锋: 5-8 个快速切换。一人一句，切换在呼吸节奏上。
- 沉默: 1-2 个镜头，极长的停留。让沉默说话。

### Seedance Prompt 策略

**5000 字符分配优先级:**
1. 微表情和反应链（40% 字符）—— 文戏的核心是人的脸上发生了什么
2. 呼吸和节奏（20%）—— 对话之间的沉默、呼吸的变化、什么时候快什么时候慢
3. 身体语言（15%）—— 手在做什么、坐姿变化、身体倾向谁
4. 运镜质感（15%）—— 手持的呼吸感、切换的节奏
5. 声音（10%）—— 房间音、衣物摩擦、呼吸声。**不是台词**

**文戏特有提示词语言:**
- "她的回答在嘴上，但她的否定在手上——右手拇指反复摩擦食指关节"
- "A 说完这句话后，B 有 1.8s 没有任何反应。不是没听见——是在决定怎么回答"
- "他看向她时，眼球运动有一个微弱的延迟——先看她的嘴，0.4s 后才看她的眼睛"

**文戏中摄像机的人格:**
- 摄像机是第三个在场者。它在观察，不是偷窥。
- 手持微呼吸感: 2px 级微动，像一个人屏住呼吸在听。
- 不飘浮、不环绕——摄像机不抢戏。演员的微表情才是特效。

### 文戏禁忌
- 不要让人物对着空气说话——视线必须有落点
- 不要"说话-切-对方-切-说话"的机械 ping-pong——反应可以在台词中间切入
- 不要让两个人物均匀地各占一半对白。权力不均衡——一个人的话更多、或更少但更重
- 不要忽略沉默。文戏中最重要的台词往往是没说的那句

## ACTION 武戏策略

武戏的核心是**身体在空间中发生了什么**。不是"打得好看"——是"这一拳为什么打出、被打的人身体哪里先着地、着地后发生了什么"。

### 故事板设计

**俯视动线图重点:**
- **空间是关键角色。** 武戏的故事板俯视图必须比文戏更详细——墙、家具、楼梯、窗户都是武器或障碍
- 人物起始位置、移动路径、每次冲突的接触点、跌倒位置、物品被撞飞后的落点
- **力矩标注**: 打击方向、受力方向、反弹方向。一拳打出去不只是"打到脸上"——是"从画面右侧挥入→接触点左颧骨→头部向左旋转→身体重心向右倾斜→右脚退半步缓冲"
- 3-5 个机位覆盖不同角度: 广角（看整体空间关系）、中景（看身体动作）、特写（看冲击瞬间）

**机位设置:**
- 广角安全机位（24-35mm）: 放在空间角落，拍下整个动作区域。这个机位保证至少有一个镜头能用。
- 手持跟拍（35-50mm）: 跟在动作后面或侧面，有呼吸感和摇晃。让观众在动作里而不是在看动作。
- 低角度冲击机位（24mm，地面高度）: 拍跌倒、撞击。地面视角让冲击有重量。
- **不要** 慢动作机位——慢动作是强调，武戏中最多用一次，在关键一击。

**镜头数策略:**
- 武戏的节奏是渐快的——开打前 2-3s/镜→打斗中 1-2s/镜→高潮 0.5-1s/镜→结束后 3-5s 静止（让观众呼吸）
- 每个镜头只包含一个动作节拍。一击 = 一镜。不要在一个镜头里塞两个打击。
- 动作和反应必须分镜——打了（镜头 A）→被打的人的反应（镜头 B）。不能在一个镜头里又打又反应。

### Seedance Prompt 策略

**5000 字符分配优先级:**
1. 空间和身体关系（30%）—— 谁在哪里、向哪个方向、用什么部位做什么
2. 物理重量（25%）—— 重力、惯性、冲量、反弹。这是武戏的"演技"
3. 动作节拍和节奏（20%）—— 精确到 0.1s 的动作时间线
4. 运镜（15%）—— 手持震动节奏、跟拍速度、冲击时的画面晃动
5. 声音（10%）—— 撞击声、脚步、呼吸。不是音乐。

**武戏特有提示词语言:**
- "这一拳不是挥——是推。从肩部出发，肘关节锁定，力量通过前臂传递到指节。接触点在对方左颧骨外侧。接触时间 0.08s"
- "被打后头部向右侧旋转——不是甩，是动量传递——旋转约 30°。同时身体重心从两脚均匀变为偏向右脚"
- "跌倒不是垂直下落——有一个 0.2s 的失控倾斜，然后右肩先着地——肩胛骨接触地面时身体还有一个向上的微反弹"
- "广角手持，距打斗 3m，镜头在每一次重击时同步微震——不是后期效果，是声波的物理震动传到操机者的手臂"

**不要写:**
- "精彩的打斗"——没有信息
- "快速的动作"——多快？什么部位？什么方向？
- "打得很好看"——这是结果不是指令
- "配合音乐节奏"——武戏的节奏在动作本身，不在音乐

### 武戏禁忌
- 不要让动作浮空——每一次发力必须有支点（脚在地面、手在墙上、身体靠着什么）
- 不要让两个人同时在做动作——总是有一个主动方和一个反应方
- 不要让受伤的人立刻恢复——伤害有持续性。被打的人之后的动作会比之前慢、比之前笨
- 不要忽略受伤后的身体状态变化——嘴角的血不是道具，是会影响之后的呼吸和表情的

## SUSPENSE 悬疑策略

悬疑的核心是**信息差**。观众知道或感觉到有东西不对——但不知道是什么、在哪里、什么时候发生。人物可能知道也可能不知道。

### 故事板设计

**俯视动线图重点:**
- 标注"已知区域"和"未知区域"——人物能看到的空间 vs 人物看不到的空间（门后、角落、身后、镜中）
- 标注"威胁方向"——威胁可能从哪里来？（用虚线箭头标注，因为人物不知道）
- 人物的移动路径必须画出来——每一步都在向未知靠近或远离未知
- 摄影机的位置 = 观众的知情权。"观众和人物看到的一样多"= 主观悬疑。"观众看到人物没看到的"= 戏剧反讽

**机位设置:**
- 人物主观视角（POV）: 50mm，手持，呼吸感。观众就是这个人。
- 偷窥视角: 长焦（85-135mm），从远处透过门缝/窗户/树叶。观众在偷看。人物不知道被看。
- 身后跟拍: 35mm，手持。观众在人物身后——"别进去"但说不出来。
- 固定广角: 24mm，locked-off。一个空的空间。什么都没有发生。但太安静了——安静本身就是威胁。

**镜头数策略:**
- 悬疑段落的镜头**越来越长**——不是越来越快。3s→4s→6s→8s。不给观众喘息——让他们盯住一个画面直到不安。
- 前 1/3: 建立空间（人物在哪里，空间里有什么）
- 中 1/3: 发现异常（一个东西不对——门开着、杯子不在原位、多了一个影子）
- 后 1/3: 接近/对峙/揭示

### Seedance Prompt 策略

**5000 字符分配优先级:**
1. 空间和未知（25%）—— 描述空间里"安静得不对"的地方
2. 微表情和身体语言（30%）—— 悬疑在脸上，不是环境里
3. 呼吸（20%）—— 悬疑场景的呼吸节奏是最重要的声音设计
4. 声音（20%）—— 有什么声音不该在？有什么声音应该在没有？安静的声音是什么质感？
5. 运镜（5%）—— 悬疑中运镜要少。不动 > 动

**悬疑特有提示词语言:**
- "她看了一眼卧室门——门缝的黑暗和刚才一样深。但有什么不一样了。说不出来。但她的身体向后倾了 2cm——身体比大脑先知道"
- "镜子里是她自己的脸。但她没有看自己——她在看自己身后的黑暗。黑暗里什么都没有。但这个'什么都没有'比'有什么'更让人不安"
- "全场最响的声音是 50Hz 冰箱嗡鸣。然后是她的呼吸。然后是呼吸暂停后的寂静。寂静持续了 4s"
- "不要在这个镜头里放任何不该出现的东西。恐怖不需要出现在画面里——恐怖只需要观众觉得它会出现"

### 悬疑禁忌
- 不要依赖音乐制造紧张——真正的不安在安静里
- 不要用跳吓——跳吓是惊吓，不是悬疑。悬疑是缓慢的、累积的、不可逆的
- 不要让威胁太早出现——"不知道"比"看到了"更恐怖
- 不要解释——门为什么开着不需要解释。杯子为什么在那里不需要解释。恐怖不需要答案

## TRANSITION 过渡策略

过渡的作用是**呼吸和连接**。它不是"凑时长"——它是两个重场之间的气口。

### 故事板设计

**俯视动线图重点:**
- 如果过渡是空间移动——比如从 A 场景走到 B 场景——画出完整的移动路径
- 如果过渡是时间流逝——用一个固定空间的空镜，标注时间变化（光线从 A 变到 B）
- 过渡段落通常只需 1-2 个机位

**过渡的类型与策略:**

| 过渡类型 | 策略 | 镜头数 | 时长 |
|---|---|---|---|
| 空间移动 | 跟拍人物从一个空间到另一个。保持连续性但降低戏剧强度——让上一个场景的情绪沉淀 | 1-2 | 4-8s |
| 时间流逝 | 固定空镜。光在空间中移动。窗外从白天变黄昏。一个静止的空间在呼吸 | 1-2 | 5-10s |
| 情绪桥接 | 一个人物的面部——她刚从上一场出来，还没进入下一场。过渡在她的脸上 | 1 | 3-6s |
| 声音过渡 | 下一个场景的声音在本段画面中提前进入（J-cut）。声音先到，画面随后 | 1 | 3-5s |
| 平行剪辑过渡 | 两个空间交替出现——A 空间正在发生的事和 B 空间正在发生的事 | 3-6 | 6-12s |

### Seedance Prompt 策略

**过渡段落的字数策略:**
- **不需要 5000 字符。** 过渡段落的目标是呼吸，不是信息密度。
- 400-800 字符通常足够——描述空间、光的变化、声音的过渡。
- 把字符留给重场段落。

**过渡特有提示词:**
- "公寓空镜。落地灯仍亮。窗外天色从深蓝变墨黑。茶几上的水杯没有人碰——水面完全静止。冰箱嗡鸣持续。远处电梯运行一次。这就是一个空间在等待。"
- "她站在公寓门外的走廊。上一场戏的情绪还在眼睛里——嘴唇仍微启 0.5mm，呼吸还没完全恢复。她摸钥匙。这个动作暂停了 1.2s——手在包里，不动。然后继续。"

### 过渡禁忌
- 不要把过渡当重场拍——过渡没有戏剧冲突，它是冲突之间的气口
- 不要在过渡段落塞信息——过渡的功能是"让上一场落地"和"为下一场准备"
- 不要跳过过渡——两个重场直接拼接会导致情绪疲劳

## PRODUCT 产品策略

产品段落的重点是**让产品成为欲望的对象**。不是"展示功能"——是"制造渴望"。

### 故事板设计

**产品段落的视觉核心:**
- 产品出现的时机——不是在开场。是在观众已经进入一个情绪/氛围/世界之后。产品是这个世界自然存在的一部分。
- 产品与人的关系——产品如何被持握、使用、与之互动。手与产品的关系是最重要的镜头。
- 材质和光——产品的材质在光下的表现（玻璃的高光、金属的拉丝、液体的折射）就是产品的"表演"

**机位:**
- 产品英雄镜头（85-135mm，f/1.4-2.8）: 浅景深，产品清晰，背景化为情绪色彩
- 手持产品镜头（50mm，手持，使用视角）: 观众在使用产品的视角
- 环境中的产品（35mm，广角）: 产品在真实世界里——不是在摄影棚

### Seedance Prompt 策略

**产品段落特有的"感官"语言:**
- 不写"产品看起来很好"——写"瓶身玻璃在侧光下折射出一条弧形的亮线，从瓶肩滑到瓶底"
- 不写"香水很好闻"——写"喷洒后腕内侧的微湿光泽在灯光下短暂闪烁，然后被皮肤吸收，只留下反射率的微妙变化"
- 声音: 瓶盖的机械咔嗒、喷洒的雾化声、产品接触表面的声音——这些声音是产品的"台词"

## ATMOSPHERE 氛围策略 (B-roll)

氛围段落是**没有叙事的叙事**。它不是推动剧情——它是建立世界的质感。

### B-roll 设计原则

**B-roll 是什么:**
- 空镜头: 空间在没有人物时的状态
- 细节镜头: 一个物体、一束光、一个质感。它们不说话，但说很多
- 环境镜头: 城市、自然、人群。人物所在的世界

**B-roll 的策略:**
- B-roll 不是一个独立的段落——它穿插在 A-roll 之间
- 在文戏对话中，在关键台词之后插入 2-3s B-roll——让那句话在观众的脑海里发酵
- 在武戏之后，插入安静的空间 B-roll——让观众从冲击中恢复
- B-roll 的运镜应该和它前后的 A-roll 不同——如果 A-roll 是手持，B-roll 可以是 locked-off。反差制造节奏

**B-roll 镜头设计:**
- 不是"拍一个杯子"——是"拍杯子外壁那颗正在下滑的冷凝水珠，85mm f/1.4，焦点在水珠上，水珠下滑的速度不均匀——它在杯壁的微纹上减速又加速"
- 不是"拍窗外的雨"——是"拍雨水在玻璃上的轨迹，一条雨线被另一条截断，两条汇合后加速流下。窗外街灯在雨滴中折射成橙色和青色的微光"
- B-roll 的每一个镜头都应该是**一张可以单独挂在墙上的照片**

**B-roll 的声音:**
- 不是音乐（大多数时候）——是环境。B-roll 让观众听到了在 A-roll 对话中被忽略的声音——雨声、风声、街道
- B-roll 的出现往往伴随声音从"对话层"切换到"环境层"——这是听觉上的呼吸

### 氛围段落的禁忌
- 不要为了凑时长加 B-roll——每一个 B-roll 镜头都必须有它存在的理由（揭示空间、提供质感、给情绪发酵的时间）
- 不要让 B-roll 变成壁纸幻灯片——每个 B-roll 有它自己的微观叙事（水的下滑、光的移动、风的节奏）

## A-roll vs B-roll 设计原则

### A-roll（叙事主线）

**是什么:** 推动剧情、揭示人物、推进冲突的段落。文戏、武戏、悬疑的核心段落都是 A-roll。

**设计原则:**
- 人物在画面中——A-roll 是关于人的
- 有动作或对话——有事在发生
- 故事板重点: 俯视动线、人物位置和移动、180 度轴线、情绪递进
- Seedance prompt 重点: 微表情、身体语言、反应链、声音设计
- 镜头之间有因果——"A 说了这句话，所以 B 做了那个反应"
- 每个镜头推动叙事——如果不能推动，这个镜头就不属于 A-roll

### B-roll（质感/呼吸/世界）

**是什么:** 建立世界的质感、提供情绪发酵的空间、连接 A-roll 段落之间的呼吸。环境、细节、氛围。

**设计原则:**
- 人物不在画面中，或人物不是焦点——B-roll 是关于空间的
- 没有"事件"在发生——但有事在"存在"
- 故事板重点: 光在空间中的移动、物体在时间中的状态、材质的微观叙事
- Seedance prompt 重点: 光的变化速度、材质的触觉质感、声音的环境层
- 镜头之间没有因果——它们是并列的，组成了一个世界的拼图
- 300-600 字符通常足够——不要用 B-roll 浪费 5000 字符

### 判断一个镜头是 A-roll 还是 B-roll

问三个问题:
1. 这个镜头里有人物吗？如果人物是焦点 → A-roll。如果人物不在或不是焦点 → 可能是 B-roll。
2. 这个镜头推动了剧情吗？如果拿掉它，故事就不完整 → A-roll。如果拿掉它，故事仍完整但少了质感 → B-roll。
3. 这个镜头和前后镜头有因果关系吗？如果"因为上一个镜头，所以这个镜头" → A-roll。如果"这个镜头和上一个镜头并列存在" → B-roll。

## 策略加载流程

在分析完用户输入/剧本后，对每一段落执行:

```
1. 识别段落戏剧功能 (DIALOGUE / ACTION / SUSPENSE / TRANSITION / PRODUCT / ATMOSPHERE)
2. 判断 A-roll 还是 B-roll
3. 从本文档加载对应策略
4. 按策略设计故事板（俯视动线、机位、镜头数）
5. 按策略写 Seedance prompt（字符分配优先级、特有语言风格）
6. 对照 film-industry-master-checklist.md 逐项检查
7. 检查 5000 字符限制
8. 交付
```

如果一个段落混合了多种功能（如悬疑 + 动作），按主导功能加载策略，从次要功能取补充元素。例如: 悬疑揭示 → 追跑（悬疑主 + 动作辅）——故事板按悬疑设计（标注未知区域、威胁方向），但机位加入动作段的广角安全机位和手持跟拍。
